# Contributing

Contributions are welcome! Bug fixes, improvements, and new features are all
appreciated.

## Before You Contribute

This project requires all contributors to sign a **Contributor License
Agreement (CLA)** before their contributions can be merged. Please read
[CLA.md](CLA.md) carefully.

By opening a pull request, you confirm that you have read and agree to the
CLA. Include the following statement in your PR description:

> I have read and agree to the Contributor License Agreement (CLA.md).

## How to Contribute

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-improvement`)
3. Make your changes
4. Test your changes locally (see README for setup instructions)
5. Commit with a clear message describing what and why
6. Open a pull request against `main`

## Guidelines

- Keep changes focused — one feature or fix per PR
- Follow the existing code style (TypeScript, React, Tailwind)
- Backend changes go in `server/index.js`, frontend in `src/`
- Test both the UI and any API changes before submitting

## Questions?

Open a GitHub issue for bugs or feature requests.
For commercial licensing inquiries, see the README.
